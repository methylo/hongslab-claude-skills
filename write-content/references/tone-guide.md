# 톤 가이드 (Tone Guide)

칼럼형과 문서형의 톤 차이, 유형별 톤 적용 규칙, 도입부 작성법을 정리한다.

---

## 칼럼형 vs 문서형 톤 비교표

| 구분 | 칼럼형 | 문서형 |
|------|--------|--------|
| **목적** | 관점 제시, 판단 기준 이동 | 정보 정리, 논리적 전달 |
| **독자와의 관계** | 함께 생각하는 동료 | 정리해서 전달하는 전문가 |
| **도입부** | 현상 관찰 또는 통념 뒤집기로 시작 | 배경·맥락 설명으로 시작 |
| **본론 전개** | 문제 재정의 → 기준 제시 → 관점 전환 | 항목별 설명 → 근거·사례 → 정리 |
| **마무리** | 관점 재정렬 (요약 아닌 질문의 전환) | 핵심 요약 + 다음 단계(CTA) |
| **문장 톤** | 단정형, 절제된 확신, 질문으로 전환 유도 | 설명형, 객관적 정리, 구조적 나열 허용 |
| **소제목 스타일** | 판단·질문형 ("왜 기존 방식이 흔들리는가") | 내용 요약형 ("AI 글쓰기의 3단계 구조") |

---

## 동일 주제 톤 비교 예시

주제: "AI 글쓰기에서 편집이 중요한 이유"

### 칼럼형

> AI로 글을 쓴다는 말이 유행이다. 그런데 대부분은 '쓴다'의 의미를 잘못 쓰고 있다. AI가 만든 텍스트는 글이 아니라 재료다. 재료를 글로 바꾸는 과정이 편집이고, **편집의 기준을 갖추지 않으면 AI는 속도만 빠른 복사기에 머문다.**
>
> 문제는 편집을 '문장 다듬기'로 좁혀서 보는 시선이다. 편집은 구조를 결정하는 행위다. 어떤 문단이 먼저 와야 하는지, 어떤 문장을 삭제해야 하는지를 판단하는 것이 편집의 본질이다. **기준 없는 편집은 수정이 아니라 교체다.**

### 문서형

> AI 글쓰기 도구는 초안 생성 속도를 단축한다. 그러나 초안에서 최종 원고까지의 편집 과정이 전체 작업 시간의 60~70%를 차지한다. 이 절에서는 편집이 왜 중요한지를 3가지 관점에서 정리한다.
>
> 첫째, 구조 편집이다. AI가 생성한 초안은 정보의 나열 순서가 독자의 이해 순서와 일치하지 않는 경우가 많다. 개념 → 기준 → 적용 순서로 문단을 재배치하는 것이 구조 편집의 핵심이다.

## 도입부 작성법 (유형별)

### 칼럼형 도입부

독자가 '당연하다'고 여기는 것을 건드리는 문장으로 시작한다. 정보를 주는 것이 아니라, 질문을 만들어야 한다.

**작성 패턴:**
```
[통념 제시] 대부분은 ~라고 생각한다.
[균열 삽입] 그런데 ~를 보면 이야기가 달라진다.
[질문 전환] 문제는 ~가 아니라 ~에 있다.
```

**예시:**
> AI로 글을 쓴다는 말이 유행이다. 그런데 '쓴다'는 표현이 정확하지 않다. AI가 만드는 것은 텍스트이고, 그것을 글로 바꾸는 것은 사람의 편집이다.

### 문서형 도입부

이 글이 다루는 범위와 독자가 얻을 것을 명시한다. 배경 설명은 3문장 이내로 끝낸다.

**작성 패턴:**
```
[배경] ~는 ~한 상황이다.
[범위] 이 글에서는 ~를 다룬다.
[기대값] 읽은 뒤 ~를 판단할 수 있다.
```

**예시:**
> AI 글쓰기 도구의 도입이 늘고 있지만, 초안에서 최종본까지의 편집 프로세스는 아직 정립되지 않았다. 이 글에서는 편집의 3단계 구조를 정리한다.

## 유형별 톤 적용 규칙

| 유형 | 톤 | 문장 특징 |
|------|-----|----------|
| 칼럼형 | 분석적 관찰자 | 단정형 서술, "~이다", 질문으로 전환 유도, 독자와 시선을 맞추는 2인칭 최소 사용 |
| 문서형 | 객관적 정리자 | 설명형 서술, "~한다", "~된다", 구조적 연결어 허용, 항목 정리 허용 |
| 뉴스레터 | 신뢰하는 동료 | 개인적 어조, 솔직함, 큐레이션 |
| 분석 기사 | 구조적 해석자 | 현상 재정의, 판단 기준 제시, 깊이 있는 사례 배치 |
| 보고서 | 객관적 전문가 | 데이터 중심, 구조적, 판단 근거 명시 |
| 보도자료 | 공식 발표 | 3인칭, 객관적, 사실 중심 |
| SNS 롱폼 | 경험을 나누는 실무자 | 스토리텔링, 교훈 도출, 공감 유도 |
